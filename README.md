# 紅白会 不動在庫マーケット

紅白薬業協同組合(紅白会)の組合員間で、医療用医薬品のデッドストックを譲渡するための試作Webアプリです。GS1コードの読み取りによる自動入力、期限切迫度に応じた価格自動計算、申込→承認の二段階フロー、譲渡書・支払集金代行依頼書の自動生成までを1つのHTMLファイルで完結します。

## GitHub Pages での公開手順

1. GitHubで新しいリポジトリを作成します(例: `kohaku-deadstock`)。
2. このフォルダの中身(`index.html`、`.nojekyll`、`README.md`)をリポジトリの**ルート**にアップロードします。
   - Webブラウザからアップロードする場合: リポジトリの「Add file」→「Upload files」でドラッグ&ドロップ。
3. リポジトリの **Settings → Pages** を開きます。
4. 「Build and deployment」の Source を **Deploy from a branch** にし、Branch を `main`(または `master`)、フォルダを `/ (root)` にして **Save**。
5. 1〜2分待つと、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

`index.html` という名前なので、上記URLを開くとそのままアプリが表示されます。`.nojekyll` はGitHubの自動変換を無効化するためのファイルです(消さないでください)。

## データの保存について(重要)

このページは**サーバーを持たない静的サイト**です。データの保存先は次のように自動で切り替わります。

- **GitHub Pages 等で開いた場合**: 各利用者の**ブラウザ内(localStorage)**に保存されます。これは「その端末・そのブラウザの中だけ」に残るデータで、**他の薬局とはリアルタイムに共有されません**。同じ端末で開けば前回のデータが残りますが、別のPCやスマホ、別ブラウザ、シークレットウィンドウでは別々のデータになります。
- 画面上部に現在の保存モードが表示されます。

つまり GitHub Pages 版は、**1人(1端末)で使うツール、または操作デモ**として動きます。**組合員全員で出品リストを共有する本番運用には、共有データベース(サーバー)が別途必要**です(下記参照)。

## セキュリティ・運用上の注意

- GitHub Pages は**誰でもアクセスできる公開ページ**です(無料プランの場合)。URLを知っていれば誰でも開けます。アプリ内のログイン・暗証番号は**画面表示を切り替えるための簡易的なもの**で、本物の認証・アクセス制御ではありません。患者情報は扱いませんが、組合員名簿・取引データを扱うため、公開URLの取り扱いにはご注意ください。
- 本格運用では、後述のサーバー版へ移行することを前提としてください。

## 本番運用(共有・認証)への発展

組合員全員で安全に共有するには、Googleアカウント認証＋共有データベースを使う構成への移植が必要です。想定する構成:

- **Googleスプレッドシート** … 出品・譲渡台帳、薬局マスタ、価格ルール、医薬品マスタ
- **Google Apps Script(GASウェブアプリ)** … この画面をほぼそのまま移植。保存先をスプレッドシートに変更し、Googleアカウントで組合員だけがアクセスできるよう制限
- **譲渡書PDFの自動生成** … 譲渡決定と同時にDriveへ保管
- **LINE WORKS通知** … 新規出品・譲渡決定を組合のトークルームへ自動通知

この構成なら、各会員の取引履歴は本人のアカウントでしか見えないよう物理的に分離でき、暗証番号運用も不要になります。

## 医薬品マスタについて

`index.html` には、MEDISの全件コードファイル(調剤包装単位・販売包装単位・元梱包装単位)から生成した約12.9万件のGTIN辞書を内蔵しています。GS1コードを読み取ると品名・規格が自動入力されます(薬価は初回のみ手入力。次回以降は学習されます)。マスタを更新したい場合は、新しい全件ファイルから辞書を作り直して差し替えてください。

## スマホのカメラでの読み取りについて

出品登録の「📷 スマホのカメラで読み取る」から、スマートフォンのカメラで医薬品のGS1コードやバーコードを読み取れます。医療用医薬品の調剤包装単位は多くが **GS1データバー限定型＋合成シンボル(CC-A)** です。読み取った内容は自動でGS1解析され、GTINから品名・規格・薬価が補完されます。

- **HTTPSが必須**です。GitHub Pages はHTTPSなのでそのまま使えます。`file://` でローカルのHTMLを直接開いた場合や `http://` ではカメラは起動しません。
- 読み取りエンジンは端末に応じて自動で最適なものを使います。①Android系: 端末標準の高精度デコーダ(BarcodeDetector)。②iPhone/Safari等: ZXing-C++(WASM版・DataMatrix対応)をCDNから読み込み。③それも不可の場合: `@zxing/browser`。②③は初回タップ時にネット接続が必要です(以降はブラウザにキャッシュ)。手入力やバーコードリーダーは従来どおりオフラインでも使えます。
- とくに **iPhoneのSafari** では、従来のJavaScript版ZXingだと読み取りが不安定でしたが、ZXing-C++(WASM)を使うことで精度が大きく向上します。
- **合成シンボル(CC-A)の注意**:GS1データバーの線形部に入っている **GTIN(商品コード)は読み取れます**(→品名・規格・薬価が自動入力)。一方、合成シンボルCC-A(2次元部)に入る **有効期限・製造番号は、ブラウザだけで動く無償のデコーダでは解読できません**。そのため、これらのコードでは **有効期限を手入力**してください(掛率による価格計算に必要です)。製造番号も必要に応じて手入力します。
- Android標準のバーコード機能(BarcodeDetector)はGS1データバーに非対応のため、本アプリは全機種で ZXing-C++(WASM)を優先的に使います。
- 初回はブラウザがカメラ使用の許可を求めます。「許可」を選んでください。背面カメラを優先的に使用します。対応端末ではライト(トーチ)のオン/オフもできます。
- **Claudeのプレビュー画面ではカメラが起動しない場合があります**(プレビューの制約)。実機での確認は、GitHub Pagesに公開したURLをスマホのブラウザで開いてお試しください。

## 動作環境

- 最新の Chrome / Edge / Firefox / Safari(16.4以降)。内蔵マスタの展開に `DecompressionStream` を使うため、古いブラウザでは品名の自動補完が動作しない場合があります(その場合も手入力で利用できます)。
- カメラ読み取りは、iOS 14.3以降のSafari、AndroidのChrome等で動作します。
